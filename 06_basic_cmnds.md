# Basic Docker commands

Docker commands interact with the **Docker Engine**, which manages images, containers, networking, and storage on the host system.

Containers run applications using **Linux kernel features like namespaces and cgroups**, which provide isolation and resource control.

---

# 1. Running Containers

### `docker run nginx`

**Command**

```bash
docker run nginx
```

**Theory**

This command creates and runs a container from the **nginx image**.

Internal workflow:

1. Docker checks if the nginx image exists locally.
2. If not found, it downloads it from Docker Hub.
3. Docker creates a writable container layer.
4. The container process starts.

**Foreground Mode**

The container attaches to the terminal, so logs appear directly in the terminal window.

**When used**

* debugging
* testing containers interactively

---

# 2. Run Container in Background

### `docker run -d nginx`

**Command**

```bash
docker run -d nginx
```

**Theory**

The `-d` flag runs the container in **detached mode**.

Detached mode means:

* container runs in background
* terminal remains free for other commands

**When used**

Production systems usually run containers in detached mode.

Example:

Running web servers, APIs, or microservices.

---

# 3. List Running Containers

### `docker ps`

**Command**

```bash
docker ps
```

**Theory**

Displays **currently running containers**.

Information shown includes:

* container ID
* image used
* command executed
* container status
* port mappings
* container name

This command is similar to the Linux process command `ps`, but specifically for containers.

**When used**

Monitoring container activity.

---

# 4. Random Port Exposure

### `docker run -d -P nginx`

**Command**

```bash
docker run -d -P nginx
```

**Theory**

`-P` exposes **all container ports to random ports on the host machine**.

Example output from `docker ps`:

```
0.0.0.0:32768->80/tcp
```

Explanation:

```
32768 → host machine port
80 → container port
```

Docker automatically selects the host port.

**When used**

* testing environments
* development environments
* avoiding port conflicts

---

# 5. Run Container with Custom Name

### `docker run -d --name nginx-cbz nginx`

**Command**

```bash
docker run -d --name nginx-cbz nginx
```

**Theory**

Containers normally get **random names generated by Docker**.

Example:

```
happy_einstein
frosty_newton
```

Using `--name` assigns a **custom name** to the container.

Benefits:

* easier management
* easier scripting
* easier monitoring

Example command using name:

```
docker stop nginx-cbz
```

# 6. Port Mapping

### `docker run -d -p 8080:80 nginx`

**Command**

```bash
docker run -d -p 8080:80 nginx
```

**Theory**

Containers run inside their own network namespace.

Without port mapping, the service is **not accessible outside the container**.

Port mapping connects:

```
host-port : container-port
```

Example:

```
8080 → host machine port
80 → nginx web server inside container
```

Users access:

```
http://server-ip:8080
```

Docker forwards traffic to container port 80.

**When used**

Publishing container services to the outside world.

---

# 7. Generic Port Mapping Format

```
docker run -d -p (external-port):(internal-port) nginx
```

**Theory**

External port → host machine port
Internal port → container service port

Example:

```
9000:80
5000:80
3000:3000
```

Used for:

* web servers
* APIs
* application servers

# 8. List Local Images

### `docker images`

**Command**

```bash
docker images
```

**Theory**

Lists Docker images stored on the local system.

Information includes:

* repository name
* tag
* image ID
* creation date
* size

Images act as **blueprints for containers**.

Multiple containers can be created from a single image.

# 9. Kill Container

### `docker kill <container-id>`

**Command**

```bash
docker kill <container-id>
```

**Theory**

Stops the container **immediately**.

Internally sends the **SIGKILL signal** to the container process.

SIGKILL:

* cannot be ignored
* immediately terminates process

**When used**

* container becomes unresponsive
* application crashes
* forced shutdown required

---

# 10. Stop Container

### `docker stop <container-id/container-name>`

**Command**

```bash
docker stop <container-id>
```

**Theory**

Stops container **gracefully**.

Docker sends:

```
SIGTERM → wait → SIGKILL
```

Application gets time to:

* save data
* close connections
* exit safely

**When used**

Graceful shutdown of services.

---

# 11. Start Container

### `docker start <container-id>`

**Command**

```bash
docker start <container-id>
```

**Theory**

Starts a container that was previously stopped.

Docker restores the container's previous configuration and filesystem.

---

# 12. Remove Container

### `docker rm <container-id>`

**Command**

```bash
docker rm <container-id>
```

**Theory**

Deletes a container from the system.

Conditions:

* container must be stopped first.

Removing container deletes:

* writable layer
* metadata

Image remains intact.

---

# 13. Force Remove Container

### `docker rm -f <container-id>`

**Command**

```bash
docker rm -f <container-id>
```

**Theory**

Forcibly removes container even if running.

Equivalent to:

```
docker kill + docker rm
```

Used in **cleanup scripts and automation pipelines**.

---

# 14. Container Size

### `docker ps -s`

**Command**

```bash
docker ps -s
```

**Theory**

Displays container storage usage.

Shows:

* writable layer size
* virtual size of container

Useful for diagnosing **disk usage problems**.

---

# 15. Remove All Containers

### `docker rm -f $(docker ps -a -q)`

**Command**

```bash
docker rm -f $(docker ps -a -q)
```

**Theory**

Removes **all containers on the system**.

Breakdown:

```
docker ps -a → list all containers
-q → show only container IDs
$( ) → command substitution
```

Commonly used during **environment cleanup**.

# 17. Run Container with Environment Variables

### `docker run -d -e MYSQL_ROOT_PASSWORD=redhat123 mysql`

**Command**

```bash
docker run -d -e MYSQL_ROOT_PASSWORD=redhat123 mysql
```

**Theory**

`-e` sets environment variables inside the container.

Used for configuring applications.

Example:

Database containers require environment variables for:

* root password
* database name
* user credentials

Example container image:

MySQL.

# 18. Execute Commands Inside Container

### `docker exec -it <container-id> bash`

**Command**

```bash
docker exec -it ad7b1a4a1470 bash
```

**Theory**

Runs a command inside a running container.

Flags:

```
-i → interactive mode
-t → allocate terminal
```

`bash` opens a shell session inside the container.

Used for:

* debugging
* checking logs
* inspecting filesystem
* running administrative commands

---

### `docker exec -it <container-id> sh`

**Command**

```bash
docker exec -it ad7b1a4a1470 sh
```

**Theory**

Some containers do not include Bash.

Example minimal Linux images:

* Alpine
* BusyBox

These use the **sh shell** instead.
